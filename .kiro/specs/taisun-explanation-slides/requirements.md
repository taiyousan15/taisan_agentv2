## 1. 目的

TAISUN AGENTシステムを素人の方にも理解してもらえるよう、専門用語に解説を付けた分かりやすいプレゼンテーション資料を作成する。

### 1.1 ビジネス目的
- システムの価値と可能性を明確に伝える
- 初心者でも迷わず使い始められるようにする
- 世界最高レベルのシステムであることを理解してもらう

### 1.2 成功指標
- 非技術者が読んで「使いたい」と感じる
- 具体的な活用事例を30個以上提示
- 手順が1-2-3で明確に示される

## 3. 用語集（専門用語解説付き）

| 用語 | 読み方 | 素人向け解説 |
|------|--------|--------------|
| **AI エージェント** | えーあい えーじぇんと | 人間の代わりに自動で仕事をしてくれるAIの「従業員」のようなもの |
| **MCP** | えむしーぴー | Model Context Protocol の略。AIが外部のツールやサービスと会話するための「共通言語」 |
| **SDD** | えすでぃーでぃー | Spec-Driven Development の略。設計書に基づいて開発する手法 |
| **要件定義書** | ようけんていぎしょ | 「何を作るか」を明確に書いた書類。家を建てる前の設計図のようなもの |

## 4. 機能要件

### REQ-001: システム概要スライド
- **種別**: 機能
- **優先度**: P0
- **要件文（EARS）**: システムは、TAISUN AGENTの全体像を1枚のスライドで視覚的に表示しなければならない。
- **受入テスト（GWT）**:
  - Given: ユーザーがスライドを見る
  - When: 概要スライドを表示する
  - Then: 82エージェント、66スキル、227 MCPツールの構成が一目で分かる

### REQ-002: MCPツール解説セクション
- **種別**: 機能
- **優先度**: P0
- **要件文（EARS）**: 各MCPツールカテゴリについて、用途と使用場面を素人向けに解説しなければならない。
- **受入テスト（GWT）**:
  - Given: 非技術者がMCPセクションを読む
  - When: 各ツールの説明を確認する
  - Then: 「どんな時に使うか」が理解できる

### REQ-003: 要件定義書作成ガイド
- **種別**: 機能
- **優先度**: P0
- **要件文（EARS）**: 要件定義書作成の手順を、Step 1、Step 2、Step 3の形式で提示しなければならない。
- **受入テスト（GWT）**:
  - Given: 初心者がガイドを読む
  - When: 手順に従って作業する
  - Then: 100点の要件定義書が作成できる

### REQ-004: 活用事例30個
- **種別**: 機能
- **優先度**: P0
- **要件文（EARS）**: 30個以上の具体的な活用事例を、業界別に分類して提示しなければならない。
- **受入テスト（GWT）**:
  - Given: ユーザーが事例を見る
  - When: 自分の業界を探す
  - Then: 具体的な活用イメージが湧く

### REQ-005: 手順付き実装ガイド
- **種別**: 機能
- **優先度**: P1
- **要件文（EARS）**: 各事例について、実装手順を1番から順番に提示しなければならない。
- **受入テスト（GWT）**:
  - Given: ユーザーが事例を選ぶ
  - When: 手順を確認する
  - Then: 何から始めれば良いか分かる

### REQ-006: 他システムとの比較
- **種別**: 機能
- **優先度**: P1
- **要件文（EARS）**: TAISUN AGENTが他のAIシステムより優れている点を、比較表で示さなければならない。
- **受入テスト（GWT）**:
  - Given: 競合システムを検討中のユーザー
  - When: 比較表を見る
  - Then: TAISUN AGENTの優位性が理解できる

## 5. 非機能要件

### REQ-900: デザイン品質
- **種別**: 非機能
- **要件文**: スライドは、アニメ風のテイストで親しみやすいデザインでなければならない。
- **受入テスト**: デザインレビューで「親しみやすい」評価を得る

### REQ-901: 読みやすさ
- **種別**: 非機能
- **要件文**: 1スライドあたりの情報量は、3秒で要点が把握できる量でなければならない。
- **受入テスト**: 3秒視認テストで主要メッセージが伝わる

### REQ-902: 専門用語の解説
- **種別**: 非機能
- **要件文**: 専門用語が出現する場合、必ず括弧内に素人向け解説を付けなければならない。
- **受入テスト**: 非技術者5人が読んで全員が理解できる
